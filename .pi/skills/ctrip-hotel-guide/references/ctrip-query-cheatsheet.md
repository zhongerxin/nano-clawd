# 携程酒店查询与材料获取（Cheatsheet）

目标：让信息采集尽量可复现、可核对。

## 1) 建议让用户提供的信息

最省事的组合：
- 目的地 + 入住/退房 + 人数
- 携程酒店列表页 URL（带日期与筛选更好）
- 或者 3–8 个候选酒店详情页 URL

如果用户无法提供 URL：让用户从携程 App/网页复制“酒店列表前 10–20 行”到聊天里（包含酒店名/价格/评分/位置简述）。

## 2) URL 优先级

1. `m.ctrip.com`（移动页通常更轻、文本更容易被抓取）
2. `ctrip.com`

提示用户：尽量把筛选条件固定（例如：价格区间、距离地铁、评分、免费取消）。

## 3) 站内搜索关键词（让用户在携程里搜）

- `{城市} 酒店 {商圈}`
- `{景点} 附近 酒店`
- `{城市} 住哪里`（用于地段决策）
- `{城市} 亲子酒店` / `{城市} 近地铁 酒店`

## 4) 常见反爬/不可抓取情况与应对

现象：`webfetch` 只返回一段脚本/空壳 HTML/提示验证。

应对（按优先顺序）：
1. 改用 `m.ctrip.com` 链接
2. 让用户复制粘贴关键字段（酒店名、价格、评分、位置、取消政策）
3. 让用户提供截图（列表页 + 详情页政策区域）

## 5) 你应当如何引用价格

- 价格必须带口径：**对应日期、房型、是否含早、是否可取消**。
- 任何未能从材料核对的价格：写“未获取/以携程实时为准”。
